# webact v0.1.38 Design Spec

## Bug Fixes

### 1. `launch --browser` error when browser already running

When the default profile's browser is already running and the agent calls `launch --browser brave` without `--profile`, return an error: `"Default browser already running (Chrome). Use --profile <name> to launch a separate Brave instance."`

Currently this is silently ignored.

## New Features

### 2. Named Profiles

Multiple browser profiles, each with its own browser process on its own port.

**Directory structure:**
```
~/.webact/
  profiles/
    default/            # protected, can't be killed
    shopping-bot/       # named by agent
    webact-a3f9/        # auto-generated
```

**Profile-scoped port files:** `~/.webact/profiles/<name>/cdp-port` replaces the global `~/.webact/chrome-port`. Each profile tracks its own browser process port.

**Launch args:**
- `launch` — uses `default` profile (current behavior)
- `launch --profile shopping-bot` — creates/reuses named profile
- `launch --profile new` — auto-generates ID (e.g. `webact-a3f9`), returns it
- `launch --browser brave --profile shopping-bot` — brave for this profile

**MCP tool:** `webact_launch` gains `profile` parameter. Output includes `Profile: <name>` so the agent can reference it later.

**Session state:** `state-<sid>.json` gains a `profile` field so commands know which profile's port file to read.

### 3. Kill/Quit Session

New command: `kill`

- Kills the current session's profile browser process
- Removes the profile's port file, profile directory, and session state files
- Errors if current session is on default profile: `"Cannot kill default profile. Use 'close' to close your tabs."`
- Default profile is protected — only custom profiles can be killed

### 4. Batch Tool

New command: `batch` (CLI) / `webact_batch` (MCP tool)

**Input:**
```json
{
  "actions": [
    {"tool": "click", "target": "--text Submit"},
    {"tool": "waitfornav"},
    {"tool": "screenshot", "output": "/tmp/result.png"},
    {"tool": "press", "key": "Escape", "wait": 500}
  ],
  "delay": 0
}
```

**Execution rules:**
- Actions run sequentially, each maps to an existing command with same arg format
- **Smart waits:** after `navigate`, `click`, `fill`, `select` — auto-wait ~500ms for network idle / DOM settle. After `screenshot`, `read`, `axtree` — no wait.
- **Per-action `wait`:** optional ms delay after that action, overrides smart default
- **Global `delay`:** optional ms between every action, additive with smart waits
- **Stop on first error:** returns all results up to and including the failed action

**Output:**
```json
{
  "completed": 3,
  "total": 4,
  "error": null,
  "results": [
    {"tool": "click", "ok": true, "output": "Clicked button \"Submit\""},
    {"tool": "waitfornav", "ok": true, "output": "--- https://..."},
    {"tool": "screenshot", "ok": true, "output": "Saved to /tmp/result.png"}
  ]
}
```

On error: `error` contains the message, failed action appears last in `results` with `ok: false`.

### 5. Grid Overlay

New command: `grid` (CLI) / `webact_grid` (MCP tool)

Injects a transparent coordinate grid overlay via JS for canvas/image-heavy apps.

**Usage:**
- `grid` — default 10x10 grid
- `grid 8x6` — 8 columns, 6 rows
- `grid 50` — 50px cell size, auto-calculates rows/cols
- `grid off` — removes overlay

**Implementation:** `eval` injects a div with `id="webact-grid-overlay"`, `position: fixed`, `pointer-events: none`, `z-index: 999999`. Each cell shows its center coordinate in small text with subtle borders. Semi-transparent background. Repeated calls replace the existing overlay.

### 6. `webact setup`

New command: `setup`

Runs MCP client detection and registration without re-downloading the binary.

**Implementation:** The install script gains `SKIP_DOWNLOAD=1` support. When set, it skips version resolution, download, and binary installation — jumps straight to MCP client configuration using `BINARY_PATH=$(which webact)`.

`webact setup` shells out to: `curl -fsSL https://webact.space/install | SKIP_DOWNLOAD=1 sh`

**Output:** Same as install script's MCP section: `Claude Code: configured`, `Cursor: already configured`, etc.

### 7. Canvas Scroll Fallback

When `scroll up/down/top/bottom` is called and the page doesn't actually scroll (checked by comparing `window.scrollY` before/after), fall back to keyboard equivalents:

| scroll arg | keyboard fallback |
|---|---|
| `down` | `PageDown` |
| `up` | `PageUp` |
| `top` | `Home` |
| `bottom` | `End` |
