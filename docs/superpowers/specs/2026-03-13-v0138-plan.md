# webact v0.1.38 Implementation Plan

## Step 1: Profile Infrastructure (lib.rs)

Modify `AppContext` path methods to support named profiles:

- `chrome_profile_dir(&self, profile: &str) -> PathBuf` — returns `~/.webact/profiles/<profile>/`
- `chrome_port_file(&self, profile: &str) -> PathBuf` — returns `~/.webact/profiles/<profile>/cdp-port`
- Keep `chrome_profile_dir()` (no args) returning default for backward compat
- Keep `chrome_port_file()` (no args) returning default for backward compat
- Add `profile` field to `SessionState` struct

## Step 2: Named Profiles in Launch (core.rs)

Modify `cmd_launch`:

- Parse `--profile <name>` arg. If `new`, generate random ID (e.g. `webact-` + 4 hex chars)
- Read port file from the profile-specific path instead of global
- If no `--profile` and browser already running and `--browser` differs from running browser: return error (bug fix #1)
- Create profile dir if it doesn't exist
- Write port to profile-specific `cdp-port` file
- Legacy migration: keep for `default` profile only

Modify `cmd_connect`:

- Store `profile` in `SessionState` when saving
- Output `Profile: <name>` in session info

## Step 3: Kill Command (core.rs + mod.rs)

Add `cmd_kill`:

- Read current session's profile from `SessionState`
- If profile is `default`: error out
- Read PID from profile's browser process (use port file + `/json/version` to confirm alive)
- Kill the browser process (send SIGTERM on unix, taskkill on windows)
- Remove: profile directory, port file, session state files for this profile
- Add `"kill"` to dispatch in `mod.rs`

## Step 4: Batch Tool (new file: commands/batch.rs + mod.rs)

Create `src/commands/batch.rs`:

- `cmd_batch(ctx, args)` — parse JSON input from first arg
- Define smart-wait commands: `navigate`, `click`, `fill`, `select`, `type`
- Loop through actions:
  - Map each action to dispatch args (same format as MCP tool args)
  - Call `commands::dispatch(ctx, tool, &args)`
  - Drain output, store in results vec
  - If error: add to results with `ok: false`, set error, break
  - Apply wait: per-action `wait` field, or smart wait (500ms for state-changing), or global `delay`
- Format output as JSON with `completed`, `total`, `error`, `results`
- Add `mod batch;` and dispatch entry in `mod.rs`

## Step 5: Grid Overlay (core.rs or new commands/visual.rs)

Add `cmd_grid`:

- Parse args: `off`, `NxM`, `N` (px size), or no args (default 10x10)
- `off`: eval JS to remove `#webact-grid-overlay`
- Otherwise: eval JS that:
  - Removes existing overlay if present
  - Gets viewport width/height
  - Calculates cell size from grid dimensions
  - Creates fixed div, z-index 999999, pointer-events none
  - For each cell: inner div with border, center coordinate label
  - Appends to body
- Add to dispatch in `mod.rs`

## Step 6: Canvas Scroll Fallback (interaction/waiting.rs or wherever scroll lives)

Modify scroll command:

- Before scrolling: capture `window.scrollY` via eval
- Execute the scroll
- After scrolling: capture `window.scrollY` again
- If unchanged and direction is up/down/top/bottom: dispatch keyboard equivalent (PageDown/PageUp/Home/End)
- Add note to output if fallback was used

## Step 7: Setup Command (core.rs)

Add `cmd_setup`:

- Shell out to: `curl -fsSL https://webact.space/install | SKIP_DOWNLOAD=1 sh`
- Capture and relay output
- Add to dispatch in `mod.rs`

Modify `install.sh`:

- At the top, after `set -e`, check `SKIP_DOWNLOAD`:
  ```sh
  if [ "$SKIP_DOWNLOAD" = "1" ]; then
    BINARY_PATH="$(command -v webact)"
    if [ -z "$BINARY_PATH" ]; then
      echo "webact not found in PATH"; exit 1
    fi
    # jump to MCP configuration section
  fi
  ```
- Restructure so the MCP config section is reachable via skip

## Step 8: MCP Integration (mcp_main.rs + tools.json)

**tools.json** — add tool definitions:

- `webact_kill`: no params
- `webact_batch`: `actions` (array, required), `delay` (number, optional)
- `webact_grid`: `grid` (string, optional — "off", "8x6", "50", default empty for 10x10)
- `webact_setup`: no params
- Update `webact_launch`: add `profile` (string, optional)

**mcp_main.rs** — `map_tool_args`:

- `kill`: no args
- `batch`: serialize actions array as JSON string, pass as first arg
- `grid`: pass grid spec as first arg
- `setup`: no args
- `launch`: map `profile` param to `--profile <value>`

**No-browser list**: add `kill`, `setup` to `no_browser` commands (they don't need auto-launch).

Add documentation for:

- Kill command: restrictions (default profile protected)

- Grid overlay: when to use (canvas apps), how to remove
